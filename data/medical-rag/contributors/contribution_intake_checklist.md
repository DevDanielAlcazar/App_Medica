# Contribution Intake Checklist

| Check | Estado | Responsable | Evidencia | Bloqueante |
|-------|--------|-------------|-----------|------------|
| Identidad verificada | Pendiente | Soporte/Admin | Documento de identidad | Sí |
| Credenciales verificadas (si aplica) | Pendiente | Admin/Médico | Cédula/certificado | Sí |
| Conflicto de interés revisado | Pendiente | Legal | Declaración COI | Sí |
| Contenido sin datos pacientes | Pendiente | Médico | Revisión manual | Sí |
| Licencia/autorización registrada | Pendiente | Legal | Permission record | Sí |
| Dominio clínico identificado | Pendiente | Médico | Metadata | No |
| Riesgo clínico clasificado | Pendiente | Médico | Risk assessment | Sí |
| Fuentes trazables | Pendiente | Médico | Bibliografía | Sí |
| Revisión médica asignada | Pendiente | Admin | Assignment | Sí |
| Revisión legal asignada | Pendiente | Admin | Assignment | Sí |
| Plan de evaluación definido | Pendiente | QA | Eval plan | Sí |
| Estado inicial asignado | Pendiente | Admin | State: draft/license_review | No |

## Decisiones

- `approved` - Entra a pipeline.
- `requires_changes` - Se solicitan modificaciones.
- `rejected` - No se acepta la contribución.

## Responsables

| Rol | Quién puede aprobar |
|-----|---------------------|
| Identidad | Admin/Soporte |
| Credenciales | Médico/Admin |
| COI | Legal |
| Riesgo clínico | Médico |
| Licencia | Legal |
| Estado inicial | Admin |
